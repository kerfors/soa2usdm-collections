Structural decisions
--------------------
1. Second label sub-column. Physical column 2 contains only the header row
   labels "VISIT" and "WEEK". Treated as part of the label area
   (property_name_source, column_position 2) and excluded from schedule_grid.
   Data columns are physical columns 3+.
2. Empty column between Visit 5 and Visit 7 (Table 1, physical column 8).
   Transcribed literally: empty header cells, no marks. No Visit 6 exists
   anywhere in the grid. Appears as a blank column in the consolidated view,
   matching the source.
3. "Study drug record / Medications dispensed / Medications returned":
   one physical row containing three text lines with a single set of marks.
   Kept as ONE activity (row 24); cell_text preserves the line breaks.
   Modelling as parent + children would require a corrections pass, since the
   marks belong to the merged row.
4. WEEK value "-.3" for Visit 2: transcribed literally.
5. "Hemoglobin A1C" (p.53) vs "Hemoglobin A1c" (p.54): subscript rendering
   difference; transcribed per page. Consolidation fuzzy-matched them into one
   unified activity (the single fuzzy match; recorded in name_variations).
6. Trailing colon on "Laboratory (Chem/Hemat):": kept in cell_text, dropped
   from the clean activity_name (both tables handled identically).
7. Multi-line labels "CT Scan (if not within last year and patient passes all
   other screens)" and "Plasma Specimen (Xanomeline)": joined with single
   spaces in activity_name; line breaks preserved in cell_text.
8. All activities are at indentation_level 0 — the grid is flat, no section
   headers or visual hierarchy.
